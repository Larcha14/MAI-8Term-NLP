# Credit Approval PoC на базе LLM и MCP

## Описание системы

Данный проект представляет собой **proof-of-concept прототип** сервиса предварительной оценки кредитной заявки.  
Система принимает на вход **текстовое описание клиента** на естественном языке, извлекает из него ключевые признаки и формирует предварительное решение о возможности одобрения заявки.

Прототип демонстрирует совместное использование:

- **LLM-сервиса** для извлечения признаков из текста;
- **MCP-сервера** с инструментами для расчёта кредитного скора и оценки риска;
- **FastAPI-сервиса**, объединяющего компоненты системы в единый HTTP API.

---

## Назначение системы

Система предназначена для демонстрации подхода к автоматизации первичной кредитной оценки клиента на основе текстового описания его характеристик.

Пример входного описания:

> Клиент 34 года, женат, высшее образование, доход 120000, работает 5 лет, просрочек не было.

На основе этого текста система:

1. извлекает параметры клиента;
2. рассчитывает условный кредитный скор;
3. оценивает уровень риска;
4. возвращает предварительное решение.

---

## Архитектура системы

Система состоит из трёх основных сервисов:

### 1. LLM-сервис
LLM-сервис развёрнут на базе **Ollama** с моделью **Qwen2.5:0.5b**.

Функции:
- обработка текстового описания клиента;
- извлечение структурированных признаков;
- преобразование неформального текста в JSON-структуру.

---

### 2. MCP-сервис
MCP-сервис реализован с использованием **FastMCP** и предоставляет два инструмента:

#### `calculate_credit_score`
Вычисляет условный кредитный скор на основе параметров клиента.

#### `assess_credit_risk`
Определяет уровень кредитного риска (`low`, `medium`, `high`) на основе признаков клиента.

---

### 3. API-сервис
API-сервис реализован на **FastAPI** и выполняет роль оркестратора.

Функции:
- принимает HTTP-запрос от пользователя;
- передаёт текст в LLM-сервис;
- получает извлечённые признаки;
- вызывает инструменты MCP-сервера;
- формирует итоговый ответ.

---

## Логика обработки запроса

Обработка запроса выполняется по следующей схеме:

1. Пользователь отправляет текстовое описание клиента в API.
2. API передаёт текст в LLM-сервис.
3. LLM извлекает признаки:
   - возраст;
   - доход;
   - наличие просрочек;
   - наличие высшего образования;
   - семейное положение;
   - стаж работы.
4. API выполняет дополнительную постобработку и валидацию признаков.
5. API через MCP-клиент вызывает:
   - `calculate_credit_score`
   - `assess_credit_risk`
6. На основе результата формируется предварительное решение:
   - `preliminarily_approved`
   - `manual_review`
   - `preliminarily_rejected`

Учитываются:
- возраст;
- доход;
- наличие просрочек;
- наличие высшего образования;
- семейное положение;

## Особенности реализации

В текущем прототипе применяются два подхода к извлечению признаков:

### LLM-извлечение

Используется для анализа текста и получения структуры признаков.

### Правила постобработки

Используются для повышения устойчивости извлечения по ряду признаков, например:

- возраст;
- доход;
- стаж;
- семейное положение;
- наличие просрочек.

Такой гибридный подход уменьшает влияние нестабильности малой языковой модели.

## Ограничения прототипа

Текущая реализация является демонстрационной и имеет ряд ограничений:

- используется малая локальная модель, поэтому качество извлечения ограничено;
- расчёт кредитного скора является условным и не основан на реальной банковской модели;
- решение носит предварительный характер и не может использоваться как финальное кредитное решение;
- для повышения качества в промышленной системе потребуются:
  - более мощная модель;
  - расширенная валидация;
  - интеграция с реальными банковскими данными и бизнес-правилами.

## Преимущества реализованного подхода

- обработка входных данных на естественном языке;
- модульная архитектура;
- разделение функций между LLM, MCP и API;
- возможность расширения набора инструментов;
- удобство контейнерного развёртывания.
